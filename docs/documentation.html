1. Overview

Family Dashboard is a locally-hosted web application designed to run on a wall-mounted display. It combines a photo slideshow with useful family widgets — all controlled through an on-screen settings panel or a phone remote.

The app runs as a Python Flask server on your local machine, serving a fullscreen web interface via Chrome. All data stays on your network — no cloud services required for core functionality.

4. Installation

Step 1: Copy Files

Copy the entire family-dashboard folder to your Windows machine (e.g., D:\family-dashboard).

Step 2: Install Dependencies

cd D:\family-dashboard
py -m pip install -r requirements.txt

Step 3: Run the App

py app.py

Open Chrome and navigate to http://localhost:5000. Press F11 for fullscreen.

Step 4: Configure

Click the gear icon (bottom-right, hover to reveal) to open Settings. Set your photo folder path and configure widgets.

Step 5: Auto-Start on Boot (Optional)

Double-click setup.bat to create a Windows startup shortcut. The app will launch automatically on login.

Or use start.bat to manually launch the server + Chrome in kiosk mode.

Step 6: Remote Control

From any device on the same network, open http://<your-pc-ip>:5000/remote to access the phone remote.

5. Google Cloud Setup

Only needed for Calendar, Commute times, and YouTube Music search. Photos, weather, news, clock, and notes work without any Google setup.

5.1 Create a Google Cloud Project

  1. Go to console.cloud.google.com
  2. Click the project dropdown → "New Project" → name it "Family Dashboard" → Create

5.2 Enable APIs

Go to APIs & Services → Library and enable:

  • Google Calendar API
  • Directions API
  • YouTube Data API v3
  • Places API (for address autocomplete)

5.3 Create an API Key

  1. Go to Credentials
  2. Click "Create Credentials" → "API Key"
  3. Copy the key — paste it in the app under Settings → Commute → Google Maps API Key
  4. Optional: restrict the key to only Directions, YouTube, and Places APIs

5.4 Create OAuth Credentials (for Calendar)

  1. Go to Credentials → "Create Credentials" → "OAuth client ID"
  2. If prompted, configure the consent screen:
    • Choose "External" → Create
    • App name: "Family Dashboard"
    • Add your email as a test user under Audience
  3. Create OAuth client: Application type = "Web application"
  4. Add redirect URI: http://localhost:5000/oauth/callback
  5. Download the JSON file → rename to credentials.json → place in D:\family-dashboard\

5.5 Connect Calendar in the App

  1. Open Settings → Google Calendar → click "Connect Calendar"
  2. Sign in with the Google account whose calendar you want to display
  3. Click "Advanced" → "Go to Family Dashboard (unsafe)" (normal for personal projects)
  4. Grant calendar read access → done

5.6 Calendar Embed (Week/Month Views)

  1. Go to Google Calendar Settings
  2. Click your calendar name under "Settings for my calendars"
  3. Scroll to "Integrate calendar" → copy the embed URL
  4. Paste in Settings → Google Calendar → "Week/Month Embed URL"

7. Architecture

File Structure

family-dashboard/
├── app.py                  # Flask backend server
├── config.json             # All settings (auto-saved)
├── credentials.json        # Google OAuth credentials (you provide)
├── token.json              # Google OAuth token (auto-generated)
├── notes.json              # Saved notes (auto-generated)
├── requirements.txt        # Python dependencies
├── setup.bat               # Windows auto-start installer
├── start.bat               # Manual launch script
├── cache/
│   ├── photo_manifest.json # Cached photo list with face data
│   └── *.jpg               # Cached RAW/HEIC conversions
├── static/
│   ├── app.js              # Frontend application logic
│   └── styles.css          # UI styles
├── templates/
│   ├── index.html          # Main dashboard page
│   └── remote.html         # Phone remote control page
└── docs/
    └── documentation.html  # This file

Data Flow

  1. Startup: Flask loads cached manifest → serves photos immediately → kicks off incremental scan in background
  2. Photo Scan: Walks folder → compares against cache → only processes new/changed files → detects faces (OpenCV) → stores focus points → saves manifest
  3. Photo Display: Browser requests /api/photo/<id> → Flask reads file from disk (NAS or local) → streams to browser. RAW/HEIC files are converted to JPEG and cached.
  4. Slideshow: Frontend cycles through manifest, renders <img> or <video> elements with object-fit: cover and object-position based on face detection data.
  5. Settings: Frontend reads/writes /api/config → Flask persists to config.json → changes apply live without page reload.
  6. Weather: Backend fetches from Open-Meteo API every 10 minutes, caches response.
  7. News: Backend fetches RSS feeds, parses with feedparser, shuffles articles from all sources.
  8. Calendar: Agenda view uses Google Calendar API (OAuth). Week/month views use Google Calendar embed iframe.

Supported File Formats

TypeExtensionsNotes
Images.jpg, .jpeg, .png, .bmp, .gif, .webp, .tiffServed directly
RAW Photos.arw, .cr2, .nef, .dng, .rafConverted to JPEG, cached
HEIC/HEIF.heic, .heifConverted to JPEG, cached
Videos.mp4, .mov, .avi, .mkv, .webmStreamed directly, auto-advance on end

Face Detection

During photo scanning, OpenCV's Haar cascade face detector analyzes each image. If faces are found, the average center point is stored as focusX and focusY percentages. The frontend uses object-position: X% Y% to center the crop on faces instead of the image center. Photos without detected faces default to center crop (50%, 50%).

Incremental Scanning

After the first full scan, subsequent scans only process new or modified files by comparing file modification timestamps against the cached manifest. Deleted files are automatically removed. The cached manifest persists across app restarts.

9. Troubleshooting

Photos not showing

  • Check the photo path in Settings — make sure it's correct and accessible
  • For NAS paths, verify you can open the path in Windows File Explorer first
  • If NAS requires credentials, make sure Windows has them saved ("Remember my credentials")
  • Check the terminal for scan errors — look for [PHOTOS] messages
  • Try clicking "Rescan" in Settings

Weather not loading or not updating

  • Check that "Show" is toggled on in Settings → Weather
  • Verify the city name is correct — try the auto-detect button (crosshair icon)
  • Open http://localhost:5000/api/weather in a browser to check the raw response
  • Changing °F/°C or city forces a fresh fetch
  • Weather refreshes every 10 minutes automatically

News ticker not visible

  • Check "Show" toggle in Settings → News
  • Verify RSS feed URLs are valid — open http://localhost:5000/api/news to check
  • The ticker may be behind other elements — check the height setting

Calendar OAuth errors

  • Make sure credentials.json is in the family-dashboard folder
  • Verify the OAuth client type is "Web application" with redirect URI http://localhost:5000/oauth/callback
  • Add your email as a test user in the Google Cloud Console → OAuth consent screen → Audience
  • If you see "Access blocked", your email isn't in the test users list

Calendar shows "Connected" but no events

  • Open http://localhost:5000/api/calendar to check the raw response
  • Check the terminal for [CALENDAR] Error: messages
  • The token may have expired — try disconnecting and reconnecting

Videos not playing audio

  • Check "Mute Video" toggle in Settings → Slideshow — toggle it off for audio
  • Chrome blocks autoplay with audio until you interact with the page — click anywhere on the dashboard once

Commute not showing

  • Requires a Google Maps API key in Settings → Commute
  • Check the schedule setting — commute may only show during configured hours
  • Set schedule to "Always" for testing

Slow photo transitions

  • Large files (RAW, high-res) take longer to load over network
  • Consider using local photos or wired ethernet to NAS
  • RAW/HEIC files are cached as JPEG after first view — subsequent loads are faster

Chrome kiosk mode not working

  • Close ALL Chrome windows first (including system tray)
  • Use F11 for fullscreen as a simpler alternative
  • Set Chrome to "Continue where you left off" in Chrome settings for persistence across restarts

'py' or 'pip' not recognized

  • Python isn't in your PATH — reinstall Python and check "Add to PATH"
  • Or use the full path: C:\Users\YourName\AppData\Local\Python\...\python.exe

DecompressionBombWarning

  • This is handled — the app disables Pillow's size limit. If you still see warnings, they're harmless.

11. Voice Commands

The voice assistant uses Google's Gemini AI to understand natural language commands and control the dashboard conversationally. Press the microphone button (bottom-right FAB bar) or press M to start a conversation.

Requires: USB microphone, Gemini API key (free from aistudio.google.com), and internet connection.

Setup

  1. Go to aistudio.google.com and sign in
  2. Click "Get API key" → create a new key (free, no credit card)
  3. In the dashboard, go to Settings → Voice Assistant → paste the API key
  4. Select model: Gemini 2.5 Flash-Lite (free) is the default
  5. Set warm timeout (how long the mic stays active between exchanges)

How It Works

  1. Press mic button or M key → voice overlay appears, mic activates
  2. Speak naturally → Chrome transcribes your speech (Web Speech API, free)
  3. Transcribed text + current dashboard data (weather, calendar, notes) sent to Gemini
  4. Gemini interprets your intent, returns a spoken response + dashboard action
  5. Dashboard speaks the response and executes the action simultaneously
  6. Mic stays warm for 15 seconds (configurable) — speak again to continue the conversation
  7. After the timeout, the conversation ends automatically
  8. Press mic button again to end early

Visual States

Mic ColorState
Red (pulsing)Listening — speak now
Blue (pulsing)Thinking — Gemini is processing
Green (pulsing)Speaking — AI is responding
YellowWarm — waiting for your next input (countdown ring shows remaining time)

What You Can Say

Unlike simple keyword matching, the Gemini-powered assistant understands natural language. All of these work:

  • "What's the weather?" / "Do I need an umbrella?" / "How cold is it tomorrow?"
  • "Play some jazz" / "Put on Blinding Lights" / "Something more upbeat"
  • "What's on my calendar today?" / "Do I have anything after 3?" / "What about tomorrow?"
  • "Add a note that the plumber is coming Thursday" / "Pin that to the screen"
  • "Next photo" / "Show me the week view" / "Close that"
  • "What time is it?" / "How tall is the Eiffel Tower?" / "Tell me a joke"

Follow-up questions work within a conversation: "What's the weather?" → "And tomorrow?" → "Will it rain this weekend?"

Conversation Memory

The assistant remembers the last 10 exchanges within a session. Context resets when the page reloads. This allows natural follow-ups without repeating yourself.

12. YouTube Music Player

The music player uses the YouTube IFrame Player API to play music videos with audio through your speakers. The video is hidden — you only hear the audio. Your YouTube Premium account works automatically since Chrome is signed into your Google account.

Setup

  1. Ensure YouTube Data API v3 is enabled in your Google Cloud Console
  2. In the dashboard, go to Settings → YouTube Music → paste your API key (same key as Google Maps)
  3. Toggle "Show" on

Using the Player

  • Click the search icon (🔍) on the music widget to open the search panel
  • Type a song or artist name — results appear with thumbnails
  • Click a result to play it — audio starts immediately
  • Use play/pause, next, previous buttons on the widget
  • Or use voice commands: "Play [song name]", "Pause", "Next song"

How It Works

  1. Search query goes to the backend → YouTube Data API v3 returns video results
  2. Selected video loads in a hidden YouTube IFrame Player
  3. Audio plays through the system's default audio output (your Bluetooth speaker)
  4. Search results become a queue — when a song ends, the next one plays automatically
The player uses YouTube video results (not YouTube Music tracks), but for most songs the audio is identical. Your YouTube Premium subscription provides ad-free playback.

13. Kiosk & Auto-Start Setup

For a permanent wall-mounted display, configure the mini PC to boot directly into the dashboard.

One-Time Setup

  1. Install Python — check "Add to PATH"
  2. cd D:\family-dashboard then py -m pip install -r requirements.txt
  3. Right-click configure-kiosk.bat → "Run as administrator" — disables screen timeout, sleep, lock screen, hibernate
  4. Auto-login: Press Win+R, type netplwiz, uncheck "Users must enter a user name and password"
  5. Run setup.bat — creates Windows Startup shortcut
  6. Reboot to test

Preventing Windows Update Restarts

Settings → Windows Update → Advanced options → Active hours → set to your full day.

14. Playback Controls & System Volume

A playback control bar appears at the bottom-center of the screen on hover (above the news/stock tickers).

Controls

ButtonAction
⏮ PreviousGo to previous photo/video
⏸ Pause / ▶ PlayPause or resume the slideshow
⏭ NextSkip to next photo/video
🔇 MuteToggle Windows system mute (mutes all audio output)
🔊 Volume SliderControls Windows system volume (affects soundbar/speakers)

Volume Control

The volume slider controls the Windows system master volume via the pycaw library. This affects all audio output — YouTube music, video playback, voice assistant responses, everything. The slider syncs with the actual system volume.

Separate from the "Mute Video" toggle in Settings → Slideshow, which only mutes <video> elements in the slideshow without affecting YouTube music or system audio.

15. Media Support & Smart Scaling

Smart Photo Scaling

  • Photos with aspect ratios close to the screen (within 1.25x) use cover — fills screen, minimal cropping, face-centered
  • Photos with extreme aspect ratios (panoramas, very tall) use contain — full photo visible, black bars
  • Portrait pairs always use contain — both photos fully visible in their half
  • Videos always use contain — full video visible

Face-Centered Cropping

OpenCV detects faces during scanning. When cover mode is used, the crop centers on detected faces. EXIF rotation is accounted for to correctly classify portrait vs landscape.

18. Remote Control (Phone)

Access from any device on the same network at http://<mini-pc-ip>:5000/remote

Features
